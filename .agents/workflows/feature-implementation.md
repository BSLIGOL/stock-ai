# 🚀 기능 구현 워크플로우

이 워크플로우는 새로운 기능을 안정적으로 구현하기 위한 5단계 표준 프로세스를 따릅니다.

## 1단계: 요구사항 분석
- [ ] 사용자 요청 사항을 분석하고 구체적인 기능 목록을 정리합니다.
- [ ] 기존 코드와의 영향도를 파악합니다.

## 2단계: 설계 (Design)
- [ ] 필요한 Entity 변경 사항 및 ERD 업데이트 내용을 정의합니다.
- [ ] REST API 엔드포인트 규격(Path, Method, Request/Response)을 작성합니다.

## 3단계: 구현 (Implementation)
- [ ] 도메인 기반 패키지(`news`, `stock` 등)를 생성하고 내부 계층을 구성합니다.
- [ ] 핵심 비즈니스 로직에는 무엇(What)을 하는지 명시하는 한국어 주석을 답니다. (Why는 프롬프트 활용)

## 4단계: 테스트 (Test)
- [ ] JUnit 5를 활용하여 단위 테스트를 작성하고 실행합니다.
- [ ] 예외 케이스(Boundary, Error)에 대한 검증을 포함합니다.

## 5단계: 리뷰 및 마무리 (Review)
- [ ] 면접관이나 동료 개발자의 시선에서 코드 개선점을 체크합니다.
- [ ] 리팩토링 및 성능 최적화(DB 인덱스 등)를 수행합니다.
- [ ] 컨벤션에 맞게 커밋합니다. (예: `feat: 주식 요약 기능 추가`)
